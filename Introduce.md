# 用户管理系统项目讲解词（5-10 分钟）

老师您好，下面我汇报一下我的用户管理系统项目完成情况。

我这个项目的目标很明确：做一个完整的前后端网络应用，而不只是单独写一个页面或者只写几个接口。  
所以我从一开始就按“可运行、可测试、可部署”的标准去做，技术栈是 React + NestJS + SQLite，最终实现了用户认证、密码重置和数据 CRUD 的完整闭环。

---

首先我先说一下整体设计思路。

前端我采用 React + TypeScript，核心页面包括：首页、登录、注册、重置密码和数据管理页。  
后端用 NestJS，分成了三个核心模块：auth、users 和 records。  
其中 auth 负责注册登录和 JWT 鉴权，users 负责用户实体和基础查询，records 负责业务数据的增删改查。  
数据库我用的是 SQLite，配合 TypeORM 来管理实体和表结构。

我在设计时重点考虑了两件事：

第一，数据安全和访问边界。  
比如 records 的所有接口都必须在登录态下访问，而且每条记录都绑定 userId，查询、更新、删除都按当前用户过滤，保证用户之间数据隔离。

第二，可维护性。  
前端用组件化方式把页面结构、认证状态管理和 API 请求拆开；后端按模块拆分 controller、service、dto、entity，后面要扩展也比较清晰。

---

接下来我介绍实现过程和核心功能。

第一部分是用户认证。

注册时会校验邮箱唯一性，密码使用 bcrypt 哈希后再入库。  
登录成功后返回 accessToken，前端把 token 存在本地，并通过请求拦截器自动带上 Bearer Token。  
然后我提供了 `/auth/me` 接口用于恢复登录态，这样页面刷新后不需要重新登录。

密码重置我也做了完整流程：  
先调用忘记密码接口生成 reset token，再通过重置接口提交 token 和新密码完成更新。  
因为这个是课程演示项目，我在演示环境里会直接返回 token，便于现场演示流程。

第二部分是业务数据 CRUD。

登录后进入数据管理页，可以新增记录、查看列表、编辑和删除。  
每次操作后前端都会重新拉取列表，保证页面展示和数据库状态同步。  
我还补了错误提示逻辑，接口失败会把错误信息反馈到页面，不会出现“点了没反应”的情况。

---

下面我说一下测试和质量保障。

后端我做了两层测试：  
一层是 AuthService 的单元测试，验证关键逻辑，比如重复邮箱注册、重置 token 生成。  
另一层是 e2e 测试，直接走 HTTP 流程，覆盖了注册、创建记录、查询、更新、删除等主链路。  
前端我用了 Vitest 和 Testing Library，覆盖了关键组件行为，比如表单提交流程和受保护路由渲染。

同时我对前后端都执行了 lint、test、build，确保代码质量和可运行性。

---

最后是部署和成果分析。

我把前端和后端都配置成可以部署到 Vercel 的形式：  
前端是标准静态部署，后端通过 serverless 入口运行 Nest API。  
这里我也在文档里明确说明了一个现实限制：SQLite 在 Vercel 上是演示方案，不适合长期生产持久化。

从结果看，这个项目已经满足课程要求：  
功能完整、技术链路打通、文档可复现、演示可执行。  
如果后续继续优化，我会优先做三件事：  
1）补充更细的权限控制；  
2）把重置密码改成邮件发送的真实流程；  
3）将数据库切换到生产级托管方案。
